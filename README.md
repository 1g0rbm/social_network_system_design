# Social Network System_Design

System Design социальной сети для курса по System Design.

## Функциональные требования

- регистрация:
  - юзернейм;
  - email;
  - пароль;
  - аватар;

- вход:
  - по паре логин и пароль;
  - через соц. сети (oauth);

- логаут;

- смена/восстановление пароля;

- редактирование профиля:
  - автар;
  - о себе;

- посты с фотографиями, небольшим описанием и геолокацией:
  - полный просмотр поста;
  - просмотр поста в ленте (краткое представление);
  - создание;
  - редактирование;
  - удаление;

- лайки и дизлайки на публикациях:
  - можно поставить либо лайк, либо дизлайк;
  - можно менять реакции;
  - лайки не публичны;

- комментарии под публикациями:
  - добавление;
  - удаление;
  - комментарии выводятся в ленте с пагинацией;
  - комментарии можно лайкать, как посты;
  - отвечать на комментарии нельзя;

- подписки на аккаунты:
  - подписаться на аккаунт;
  - отписаться от аккаунта;
  - просмотреть список своих подписок;
  - просмотреть список своих подписчиков;

- писать посты, комментарии и ставить лайки могут только зарегистрированные пользователи;

- поиск постов по локациям:
  - сортировка: по дате добваления, по популярности;
  - просмотр постов по локации;
  - топ мест за день/месяц/год;

- лента постов пользователей:
  - в обратном хронологичесокм порядке;
  - выводится с пагинацией;

- просмотр постов выбранного пользователя:
  - в обратном хронологичесокм порядке;
  - выводится с пагинацией;

- все списки выводятся с пагинацией;

## Нефункциональные требования

- DAU - 10 000 000;
- Аудитория - страны СНГ;
- Данные хранятся всегда;

- Активность пользователей:
  - в среднем каждый пользователь добавляет 1 пост в котором может быть до 5 фотографий раз в неделю;
  - в среднем каждый пользователь добавляет 2 комментария в сутки;
  - в среднем каждый пользователь читает по 10 комментариев в сутки;
  - в среднем каждый пользователь смотрит по 20 постов в сутки в ленте;
  - в среднем каждый пользователь открывает 3 поста в сутки;
  - в среднем каждый пользователь ставит 10 реакций в сутки;

- лимиты:
  - пользователь может прикрепить к посту не более 5 фотографий;
  - длинна поста не может быть более 1000 символов;
  - пользователь может поставить не более 1000 реакций в сутки;
  - пользователь может опубликовать не более 100 постов в сутки;
  - пользователь может написать не более 500 комментариев в сутки;

- сезонность летом и в новогодние праздники нагрузка может возрастать в 3 раза;

- тайминги:
  - создание поста не более 3с;
  - загрузка одной фотографии не более 5с;
  - добавление лайка менее 1с;
  - добавление комментария не более 1с;
  - после публикации в ленте пост появляется не позднее чем через 30с;
  - комментарий после публикации появляется не позднее, чем через 10с;
  - api latency p95 < 500мс;

- доступность  99,95%;

## Доменная модель

### Веса типов для расчетов

uuid - 16b
timestamp - 8b
int (int8) - 8b
float64 - 8b
enum(like, dislike) - 4b
string(text) 4 b + 500 слов * 2b = 1000b
string(varchar)  4  + 50b * 2b = 100b

- User (1164 bytes):
  - id uuid
  - username string
  - email string
  - password_hash string
  - avatar_url string
  - bio string
  - created_at timestamp
  - updated_at timestamp
  - deleted_at timestamp

- Post (1180 bytes):
  - id uuid
  - author_id uuid
  - title string
  - text string
  - location_id uuid
  - like_count int
  - view_count int
  - created_at timestamp
  - updated_at timestamp
  - deleted_at timestamp

- Media (240 bytes):
  - id uuid
  - filepath string
  - mime_type string
  - wigth int
  - heifth int
  - created_at

- PostMedia (32 bytes)
  - id_post uuid
  - id_media uuid

- Comment (1080 bytes):
  - id uuid
  - post_id uuid
  - author_id uuid
  - text string
  - like_count int
  - created_at timestamp
  - updated_at timestamp
  - deleted_at timestamp

- PostReaction (36 bytes):
  - user_id uuid
  - post_id uuid
  - type enum(like, dislike)

- CommentReaction (36 bytes):
  - user_id uuid
  - comment_id uuid
  - type enum(like, dislike)

- Location (1132 bytes):
  - id uuid
  - title string
  - description string
  - lat float64
  - lon float64

- LocationMedia(32 bytes)
  - id_location uuid
  - id_media uuid

- Follow (40 bytes)
  - follower_id uuid
  - followee_id uuid
  - created_at timestamp

## Оценка нагрузки

### Посты
write rps = 10 000 000 / 86400 / 7 = 16
read rps = 10 000 000 * 20 / 86400 = 2_314

write traffic = 16 * 1180 = 18_880 b
read traffic = 2314 * 1180 = 2 _730_520 b

### Комментарии
write rps = 10 000 000 * 2 / 86400  = 230
read rps = 10 000 000 * 10 / 86400 = 1200

write traffic = 230 * 1080 = 248_400 b
read traffic = 1200 * 1080 = 1 _296_000 b
